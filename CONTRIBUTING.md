# 为本项目做贡献

感谢您对本项目的兴趣！我们欢迎任何形式的贡献。

## 如何贡献

-   **报告问题**：如果您在阅读或使用指南时发现任何错误或不清晰之处，请通过 [【飞书 —— hello-vibe 阅读反馈表】](https://jspi-311.feishu.cn/sheets/PtqZseA1EhGjqgtOcwbcu1Mrn4c) 提交您的问题。请尽可能详细地描述问题。
-   **功能建议**：如果您有任何关于改进本指南的建议，请通过 [【Github Issues】](https://github.com/jspi-fu/hello-vibe/issues) 页面告诉我们。

## 提交更改 (Pull Request)

如果您想直接贡献内容：
1.  Fork 本仓库。
2.  创建一个新的分支 (`git checkout -b feature/YourAmazingFeature`)。
3.  进行您的修改。
    - 请确保您的改动符合 [【行为准则】](CODE_OF_CONDUCT.md)。
    - 在进行大规模改动或新内容撰写后，使用 [【mkdocs-shadcn】](https://github.com/jspi-fu/customized-skills/blob/master/Skills/mkdocs-shadcn/SKILL.md) 技能进行格式化。
        - 下载方式：命令行输入 `npx skills add https://github.com/jspi-fu/customized-skills --skill mkdocs-shadcn`
        - 使用方式：在 IDE/CLI 对话界面输入 `对于 @【您贡献的内容】：按照mkdocs-shadcn的风格进行格式化与智能排版`
4.  提交您的更改 (`git commit -m 'feat: Add some AmazingFeature'`)。
5.  将分支推送到您的 Fork (`git push origin feature/YourAmazingFeature`)。
6.  创建一个新的 Pull Request。

如果您对 Github PR 规范操作不熟悉，可以参考文章 [【如何在 Github 上规范的提交 PR（图文详解）】](https://zhuanlan.zhihu.com/p/584834288)
